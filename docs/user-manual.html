시스템 개요

EGTRONICS COMS(Charge Order Management System)는 EV 충전기의 수주 → 생산 → 출하 → AS 전 과정을 한 곳에서 관리하는 사내 업무 시스템입니다.

PC 브라우저와 태블릿·모바일 모두에서 사용할 수 있으며, 부서별 역할에 따라 접근 가능한 화면이 다르게 설정됩니다.

주요 업무 흐름

영업 오더 등록
→
생산 착수
→
생산 완료·검사
→
출하
→
AS 접수·처리

오더 상태 정의

생산대기 (PENDING)
생산진행중 (IN_PROGRESS)
출하대기 (AWAIT_PICKUP)
출하완료 (COMPLETED)

역할 및 권한

COMS는 5개 역할을 지원합니다. 로그인 후 역할에 따라 상단 탭 메뉴가 자동으로 조정됩니다.

역할 부서 접근 가능 화면
관리자 충전기개발실 모든 화면 + 사용자 관리
영업 영업부 영업 입력, 생산 대기, 조회, AS 접수
생산 생산부 / 생산2팀 생산 대기, 생산 입력, 출하대기, 조회
품질 품질관리본부 출하대기, 조회, AS 접수, AS 처리
A/S 기술지원팀 조회 (AS 이력 포함)

로그인

COMS는 이메일 인증 방식으로 로그인합니다. 초기 비밀번호는 관리자에게 문의하세요.

1
아이디(ID) 입력
관리자로부터 발급받은 아이디를 입력합니다.
2
비밀번호 입력 후 로그인
비밀번호를 입력하고 로그인 버튼을 누릅니다.
3
역할별 시작 화면으로 이동
로그인 성공 시 역할에 맞는 기본 화면으로 자동 이동됩니다. 화면 우측 상단에 이름·역할이 표시됩니다.
ℹ️
비밀번호를 분실한 경우 관리자(박우진)에게 초기화를 요청하세요. 이메일 인증 코드 발송도 지원합니다.

1영업 입력

관리자 영업

고객으로부터 수주한 EV 충전기 오더를 신규 등록합니다. 등록 즉시 생산 대기 큐에 추가됩니다.

화면 구성
영업 부서 · 신규 오더 등록
섹션입력 항목필수
① 발주 정보고객사, 고객 담당자, 충전기 용도(공용/비공용), 모델, 케이블 길이, 납품일자, 충전소 ID, 충전기 번호, CPO 운영사고객사·모델·케이블·납품일
② 통신 모듈라우터번호(S/N), USIM번호(ICCID)—
③ 설치 정보설치주소, 현장 담당자 이름·전화번호—

고객사 / 담당자 추가

목록에 없는 고객사는 + 추가 버튼으로 즉시 마스터에 등록할 수 있습니다. 담당자도 동일하게 관리 버튼을 통해 추가합니다.

입력 미리보기

화면 우측의 입력 미리보기 패널이 실시간으로 입력 값을 요약 표시합니다. 등록 전 내용을 최종 확인할 때 활용하세요.

CSV 업로드

다수 오더를 한 번에 등록할 때는 CSV 업로드 버튼을 사용합니다. 템플릿 파일은 docs/order_test.csv를 참고하세요.

1
고객사 선택 또는 입력
드롭다운에서 기존 고객사를 선택하거나, 새 이름을 직접 입력 후 + 추가를 누릅니다.
2
충전기 용도 선택
공용: 환경부 CPO 등록 대상 · 검정일자 입력 필요 / 비공용: 독립 운영
3
모델 및 케이블 길이 선택
드롭다운에서 모델 라인업을 선택하고, 케이블 길이(3m / 5m / 7m / 10m 또는 직접 입력)를 지정합니다.
4
오더 등록
우측 상단 오더 등록 버튼을 누릅니다. 오더 ID가 자동 채번되고 생산 대기 목록에 추가됩니다.
⚠️
충전소 ID는 영문·숫자 조합이며, 환경부 ME카 또는 자체 관제 ID(예: CT3006, ME0001) 형식을 따릅니다.

2생산 대기

관리자 영업 생산

등록된 오더의 생산 현황을 확인하고 오더를 선택해 생산 입력 화면으로 이동합니다. 4가지 뷰(칸반·카드·타임라인·테이블)를 지원합니다.

뷰 전환

칸반
카드
타임라인
테이블
뷰특징추천 상황
칸반신규접수 / 긴급 / 작업중 / 출하대기 컬럼으로 상태별 분류생산팀 현황판 · 상태별 빠른 파악
카드오더를 카드 형태로 격자 배치, 수정됨·긴급 태그 표시태블릿·모바일에서 터치 접근
타임라인납품일 기준 시간순 정렬, 날짜 강조납기 준수 여부 한눈에 파악
테이블오더#·고객사·모델·충전소ID·주소·납품일·상태 컬럼다수 오더 비교·정렬·검색

D-day 색상 규칙

  • 빨간색 (D-7 이하) — 긴급. 즉시 조치 필요
  • 주황색 (D-8 ~ D-14) — 주의. 생산 착수 확인
  • 검정 (D-15 이상) — 여유

검색 및 필터

  • 상단 검색창에서 고객사·모델·충전소 ID·오더번호로 검색
  • 모델 드롭다운으로 특정 모델만 필터링 가능
  • 초기화 버튼으로 검색 조건을 모두 지웁니다
💡
영업 담당자는 생산대기 목록에서 수정됨 태그가 붙은 오더를 클릭해 발주 정보를 수정할 수 있습니다.

3생산 입력

관리자 생산

생산 대기 목록에서 오더를 선택하면 이 화면으로 진입합니다. 생산 실적 7개 항목을 입력하고 기능 검사 성적서를 작성합니다.

생산 실적 입력 항목
#항목설명
1생산일자 *공장 조립 및 최종 자체 검사 완료일
2로트번호 *생산일자 기준 자동 제안, 수정 가능 (예: L26-W28-A)
3시리얼 *외형/제어보드 고유 시리얼 — 추천값 클릭으로 자동 입력, 중복 확인 내장
4검정일자 *KTC 등 공인기관 형식승인·검정 완료일 (유효기간 7년)
5S/W 버전 *버전 칩에서 클릭 선택 또는 직접 입력
6F/W 버전 *버전 칩에서 클릭 선택 또는 직접 입력 (BETA 버전 별도 표시)
7기능 검사 성적서 *작성하기 버튼으로 체크리스트 작성 — 출하대기 등록 전 필수

생산 단계 흐름

생산대기(PENDING)
→
생산 시작 클릭
→
실적 입력(IN_PROGRESS)
→
출하대기 등록
→
AWAIT_PICKUP
⚠️
기능 검사 성적서를 작성하지 않으면 출하대기 등록 버튼이 비활성화됩니다. 반드시 체크리스트를 완료한 뒤 저장하세요.

수정 이력

영업 담당자가 발주 정보를 변경한 경우 수정이력 N건 배지가 표시됩니다. 클릭하면 변경 내용을 확인할 수 있습니다.

4출하대기

관리자 생산 품질

생산·검정이 완료되어 출하 대기 중인 오더를 관리합니다. 출하 전 검사 성적서 확인 후 출하 완료 처리합니다.

상단 통계 카드
카드내용
총 출하대기기능 검사 성적서 발급 완료 건수
금주 완료이번 주(월~오늘) 출하 완료 건수
평균 리드타임영업 접수 → 생산 완료 평균 일수
평균 검정 소요생산 → 검정 완료 평균 일수

출하 처리 절차

1
기능 검사 성적서 확인
목록의 기능검사성적서 버튼을 클릭해 생산팀이 작성한 내용을 검토합니다.
2
출하 전 검사 작성
출하검사 N항 버튼을 클릭해 출하 전 체크리스트를 작성합니다. 사진 첨부도 가능합니다.
3
출하 완료
출하 완료 버튼을 클릭하면 상태가 COMPLETED로 전환되고 조회 화면으로 이동됩니다.
💡
CSV 내보내기 버튼으로 출하대기 목록 전체를 Excel 파일로 다운로드할 수 있습니다 (한국어 UTF-8 BOM 포함).

5오더 조회

관리자 영업 생산 품질 A/S

출하 완료된 오더를 검색하고 전체 이력을 조회합니다. A/S 역할은 AS 접수·처리 이력도 함께 확인할 수 있습니다.

검색 방법

  • 검색창에 고객사 · 충전소 ID · 시리얼 · 주소 · 납품일 등 키워드를 자유롭게 입력
  • 고급 필터로 모델, 날짜 범위, 상태를 세분화
  • 상단 출하완료 오더만 표시 토글로 전체/완료만 전환

상세 정보 확인

목록 행을 클릭하면 우측에 상세 패널이 열리고 기본 정보 · 생산 정보 · 성적서 · 변경 이력 탭을 확인할 수 있습니다.

6AS 접수

관리자 영업 품질

현장에서 발생한 충전기 고장·이상을 접수합니다. 접수번호는 AS-{연도월일}-{순번} 형식으로 자동 채번됩니다.

접수 목록 상태 탭
탭설명
전체모든 AS 접수 건
접수대기접수 후 담당자 미배정 상태
담당자배정담당자가 배정되어 출동 예정
처리중현장 처리 진행 중
처리완료조치 완료 및 종결

새 접수 등록

1
새 접수 등록 버튼 클릭
우측 상단 + 새 접수 등록을 클릭합니다.
2
고객사·충전소·고장 유형 입력
고객사, 충전소 ID, 고장 유형, 긴급도(일반/긴급), 접수자 정보를 입력합니다.
3
등록
저장하면 접수번호가 자동 발번되고 상태가 접수대기로 설정됩니다.

목록의 처리 → 버튼을 누르면 AS 처리 화면으로 이동해 담당자 배정 및 조치를 진행합니다.

7AS 처리

관리자 품질

좌측 목록에서 AS 건을 선택하면 우측에 처리 패널이 열립니다. 담당자 배정, 상태 변경, 조치 내용 입력, 사진 첨부까지 한 화면에서 처리합니다.

처리 패널 탭 구성
탭내용
처리 현황담당자 배정, 출동 예정일, 처리 상태, 조치 유형, 발생 비용, 조치 상세 내용 입력
접수 정보최초 접수 시 입력한 고장 내용·고객 연락처 등 원본 정보
처리 이력상태 변경 이력 (타임라인 형식)
사진현장 사진 첨부 및 조회

조치 유형 분류

  • 하드웨어(HW) 및 부품 관리 — 부품 교체, 배선 교체, 외형 교체, 하드웨어 교체, 제품교체
  • 소프트웨어(SW) 및 설정 관리 — 펌웨어 업데이트, HMI 업데이트, 설정 변경, 원격 리셋
  • 통신 및 네트워크 — 통신연결불량, 무선모뎀 문제
  • 인프라 설치 및 외부 요인 — 설치 문제, 차량 이상 진단, 사용자 사용 문제
  • 원인 분석 및 기타 — 원인분석, 특이사항없음
ℹ️
상태를 처리완료로 변경하면 처리 이력에 완료 시각이 자동 기록됩니다. 완료 후에도 사진 추가 및 내용 수정이 가능합니다.

8사용자 관리

관리자 전용

시스템 계정을 추가하고 역할·권한을 설정합니다. 관리자(admin) 역할만 접근할 수 있습니다.

역할 카드

화면 상단에 역할별 카드가 표시되며, 각 카드에 접근 가능한 탭 목록과 현재 인원 수가 요약됩니다.

사용자 추가

1
사용자 추가 버튼 클릭
우측 상단 + 사용자 추가를 클릭합니다.
2
정보 입력
이름, 아이디(ID), 역할, 부서, 전화번호, 이메일을 입력하고 임시 비밀번호를 설정합니다.
3
저장
저장 즉시 해당 계정으로 로그인이 가능합니다. 사용자에게 아이디와 임시 비밀번호를 전달하세요.

사용자 수정 / 삭제

각 행의 ✏️ 편집 버튼으로 정보를 수정하고, 🗑️ 삭제 버튼으로 계정을 삭제합니다. 자기 자신의 계정은 삭제할 수 없습니다.

자주 묻는 질문

Q. 오더 등록 후 수정이 가능한가요?

영업 역할은 오더가 PENDING · IN_PROGRESS 상태일 때 발주 정보를 수정할 수 있습니다. 수정 시 생산 화면에 수정이력 N건 배지가 표시됩니다.

Q. 시리얼 번호 추천값은 어떻게 계산되나요?

모델 코드 + 연도/월 알파벳 코드(2023년=A, 이후 +1) + 로트 내 순번으로 자동 생성됩니다. 동일 시리얼이 이미 존재하면 중복 경고가 표시됩니다.

Q. 기능 검사 체크리스트는 어디서 관리하나요?

docs/func/{모델코드}.json 파일로 모델별 체크리스트를 관리합니다. 파일이 없으면 시스템 기본 체크리스트가 사용됩니다.

Q. 출하 전 사진은 어떻게 첨부하나요?

출하대기 목록에서 성적서 보기 → 출하 검사 드로어의 사진 첨부 영역에서 파일을 선택하면 Supabase Storage(ship-photos 버킷)에 자동 업로드됩니다.

Q. CSV 내보내기 시 한국어가 깨집니다.

파일을 Excel에서 열 때는 데이터 → 텍스트/CSV에서 가져오기를 사용하고 인코딩을 UTF-8로 선택하세요. 직접 파일을 열면 인코딩이 자동으로 인식되지 않을 수 있습니다.

Q. 비밀번호를 잊었을 때는?

관리자(박우진, admin)에게 비밀번호 초기화를 요청하세요. 사용자 관리 화면에서 즉시 처리가 가능합니다.
